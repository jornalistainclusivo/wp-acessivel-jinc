# JINC WP Acessível - API & Code Reference

This document provides a comprehensive overview of the PHP architecture and the internal APIs available in the `src/` directory. The plugin follows strict **Domain-Driven Design (DDD)** principles and is engineered for high performance, utilizing `strict_types=1` across all files.

## Core

The `Core` namespace acts as the central initialization layer.

* **`JINC\Core\Bootstrap`**: Entry point for the plugin. Registers all modules and utility classes.
* **`JINC\Core\Gatekeeper`**: Security and environmental validation layer (e.g., ensuring PHP 8.1+ and WordPress 6.0+).

## Modules

The plugin features are isolated into independent modules.

### SemanticEnforcer (Phase 1)

Responsible for real-time HTML parsing and semantic fixing using `DOMDocument`.

* **`ContentFilter`**: Hooks into `the_content` and coordinates the parsing and mutation steps.
* **`DOMSerializer`**: Handles safe loading and saving of HTML5, ensuring `UTF-8` encoding and suppressing libxml errors.
* **`HeadingHierarchyFixer`**: Interrogates the DOM and repairs incorrect `H1-H6` sequences.
* **`LandmarkInjector`**: Automatically wraps orphan blocks with semantic ARIA landmarks (`<main>`, `<article>`).
* **Supporting Classes**: `HeadingAnalysis`, `HeadingInfo`, `HeadingViolation`, `LandmarkAnalysis`.

### MediaGatekeeper (Phase 2)

Operates on the REST API to ensure no images bypass accessibility requirements.

* **`RestUploadValidator`**: Hooks into `rest_pre_insert_attachment` to block uploads missing alt text.
* **`AltTextValidator`**: Evaluates alt text rules (e.g., rejecting generic phrases like "image of" or enforcing the `_jinc_decorative` flag).
* **`AltTextStatus` / `AltTextValidationResult`**: Value objects for validation outcomes.
* **`AttachmentMetaFilter`**: Manages the storage of custom meta keys (e.g., `_jinc_decorative`).
* **`AdminNoticeManager`**: Renders UX feedback in the wp-admin for blocked uploads.
* **`DescreveAIClientInterface` / `NullDescreveAIClient` / `DescreveAIResult`**: Interfaces for the forthcoming AI integration (Phase 4) that generates alt text automatically.

### FrontendBar (Phase 3)

Injects the accessibility UI onto the frontend.

* **`BarInjector`**: Hooks into `wp_body_open` (and `wp_footer` as a fallback) to render the accessibility toolbar HTML, and enqueues `jinc-bar.css` and `jinc-bar.js`.

### ThemingEngine (Phase 3)

Manages the settings and overrides for the visual presentation.

* **`SettingsPage`**: Registers the WordPress Settings API options (`jinc_theme_options`) and renders the backend configuration page.
* **`ThemeEngine`**: Injects custom CSS variables into the frontend based on the configured options to ensure high contrast and WCAG AAA compliance.

## Utils

Shared technical services used across the domains.

* **`CacheManager`**: Abstraction over the WordPress Transients API (ADR-003) to cache the output of `DOMDocument` processing.
* **`DOMDocumentHelper`**: Utility functions for manipulating DOM nodes safely.
* **`Logger`**: Structured logging for debugging and telemetry.
