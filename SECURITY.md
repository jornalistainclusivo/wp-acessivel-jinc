# Security Policy

## Supported Versions

Currently, we only support the latest version of **WP Acessível JINC**. We strongly recommend keeping your WordPress installation and the plugin up to date.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Responsible Disclosure Policy

Security is a top priority for the Jornalista Inclusivo (JINC) team. We appreciate the work of the security research community and encourage the responsible reporting of any vulnerabilities found in our software.

**Please DO NOT open a public issue on GitHub for security vulnerabilities.**

If you believe you have found a security vulnerability, please report it to us via secure email at:
**[Inserir E-mail de Segurança Aqui - ex: security@jornalistainclusivo.com]**

### Service Level Agreement (SLA)

We take all security reports seriously and aim to address them promptly:

* **Triage & Initial Response:** Within **48 business hours** of receiving your report, you will receive an acknowledgment from our security team.
* **Resolution & Patching:** We prioritize critical vulnerabilities and aim to release patches as quickly as possible. The timeline depends on the severity and complexity of the issue.

### Guidelines for Researchers

When submitting a vulnerability report, please include:
1. A detailed description of the vulnerability.
2. Steps to reproduce the issue (including any necessary configuration).
3. Potential impact and any proof-of-concept (PoC) code or screenshots.

We ask that you:
- Do not exploit the vulnerability further than necessary to demonstrate the issue.
- Do not disclose the vulnerability publicly until a patch has been released and users have had reasonable time to update.
- Respect our users' privacy and data; do not interact with accounts you do not own.

Thank you for helping us keep WP Acessível JINC secure!
